# Inbox Zero

## Overview
"Inbox zero" isn't about having literally zero emails — it's about having zero *undecided* emails. Every message in your inbox either has a decision made or is deliberately saved. The goal: inbox stops being a todo list that grows on its own.

## When to Use
- Inbox has hundreds of unread messages
- You've been avoiding email and it's accumulating
- You want to process today's email in one batch
- Starting a new habit of processing email 1-2x daily instead of continuously
- Returning from vacation with a flood

## Process

### 1. Sort by sender first (one-time cleanup)
For a full reset on a bloated inbox:
- Sort by sender
- Bulk-archive newsletters you never read (unsubscribe later)
- Bulk-archive automated notifications (CI, calendar reminders, system alerts)
- Leave only human messages for the 4D pass

Most cleanup is cutting noise, not responding.

### 2. The 4D for every remaining email
Touch each email **exactly once**. Decide:

**D1. Delete / Archive**
- Newsletters you don't read → unsubscribe now, not "later"
- FYI messages you've absorbed → archive
- Threads that resolved themselves → archive

**D2. Delegate**
- Can someone else handle this? Forward with a clear ask + deadline.
- Close the loop: archive after forwarding, don't wait for reply to archive.

**D3. Defer**
- Needs response but not now? Two-minute rule: if it takes >2 min, defer.
- Put it on a calendar block ("Tuesday 3pm: email batch") or in a task system.
- Don't leave "I'll get to it" in your inbox. That's how it stagnates.

**D4. Do**
- If it takes < 2 min, do it now. Reply. Forward. Click the button.
- If it's a quick reply, don't overthink the wording.

### 3. Use folders sparingly
One archive folder > ten categorized folders. Search beats sorting.

Exception: if you regularly pull up a category (e.g. "receipts" for expenses), that's worth a folder or a label.

### 4. Templates for repeat replies
Keep canned responses for things you type often:
- "Got it, will have a response by [day]"
- "Not the right person — please reach out to [X]"
- "Let me think on this and get back to you by [day]"

Save in Lark / Gmail / Superhuman canned responses. Cuts 30s off each response.

### 5. Set processing windows, not continuous checking
Pick 2-3 times per day:
- Morning (after first deep work block)
- Post-lunch
- End-of-day wrap

Outside these windows: email is closed. Notifications off. Auto-responder if needed.

### 6. For drafts — write once, check once, send
No more re-opening and re-editing. Write it, re-read it for tone and typos, send. If you want to delay (send tomorrow), schedule-send.

### 7. The weekly drain
Once a week (Friday EOD), drain anything still lingering:
- Anything > 7 days old that you won't reply to → archive with a one-line acknowledgment, or delete
- If you've been "deferring" the same email 5 times, just decide: do it now or don't do it

Backlog beyond 7 days is usually dead. Let it go.

## Rationalizations

| Excuse | Reality |
|--------|---------|
| "I need to keep it as a reminder" | Your inbox is a terrible reminder system. Use a task tool |
| "I might need to reference this later" | Archive + search works. "Unread = unread" doesn't |
| "I'll reply when I have more time" | More time isn't coming. 2-min or defer to calendar block |
| "I check email constantly because urgent things happen" | If urgent, they'll call. Email ≠ real-time |
| "I have 800 unread and it's fine" | It's not fine. Things slipped. Admit it, clean it, move on |

## Red Flags
- Email tab open continuously during deep work
- Using inbox as todo list (stars, unread, flags)
- Same thread replied to 6+ times for clarity → should have been a call
- Chronic "apologies for the late reply" opener → your system is broken, not your character
- Vacation = guilt because of "the inbox situation"
- Inbox count > 50 after a full processing session

## Verification
- [ ] Inbox reduced to under 20 actionable items
- [ ] 2-3 email processing windows scheduled daily
- [ ] Email closed / notifications off outside those windows
- [ ] At least one recurring newsletter unsubscribed (weekly)
- [ ] No message starred "to do later" for more than 5 days
- [ ] Friday drain done this week

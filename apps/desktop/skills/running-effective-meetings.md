# Running Effective Meetings

## Overview
Meetings are the most expensive coordination tool an org has. Poorly-run meetings silently tax every attendee. The strongest principle: default to *no meeting*, and when a meeting is needed, make it earn its time.

## When to Use
- Preparing to call a meeting (does it need to be a meeting?)
- Designing an agenda before a scheduled meeting
- Facilitating a live discussion
- Writing meeting notes / action items after
- Reviewing whether a recurring meeting should continue

## Process

### 1. Pre-meeting — earn the invite
Ask the 3 kill-switch questions:
- **Purpose**: What decision needs to be made, or what alignment created?
- **Decider**: Who is the one accountable decision-maker?
- **Pre-read**: Could this be an async doc + comments instead?

If any answer is weak, downgrade: 1:1 Slack message, async doc, or Loom recording.

### 2. Agenda design (if proceeding)
- **Outcome line at the top**: "By the end, we will have decided X / aligned on Y"
- **Time-boxed items**: each topic has minutes allocated
- **Pre-read attached**: context for any decision item
- **Optional attendees marked**: people who can read notes instead
- **Questions, not topics**: "Should we launch in Q2?" beats "Q2 launch"

### 3. Facilitation — the 90/10 rule
- Spend 10% on setup (recap outcome, acknowledge attendees), 90% on substance
- Enforce time-boxes. Park rabbit holes to a "parking lot" doc
- Silent-writing break for hard decisions (everyone writes for 2min, then share)
- Name the decider before the decision: "Alice, you'll call this one"

### 4. Close — the last 5 minutes
Every meeting ends with:
- **Decisions made** (1-3 bullets)
- **Action items**: owner + deadline for each
- **Parking lot items**: saved for later, not lost
- **Next meeting?**: is a follow-up actually needed?

### 5. Notes distributed within 24h
Short note (decisions + actions) to all attendees + interested observers.

## Rationalizations

| Excuse | Reality |
|--------|---------|
| "Everyone needs to be on this call" | Optional attendees + async notes respect people's time |
| "We'll figure out the agenda live" | 60% of meeting value comes from the agenda |
| "We can follow up on action items later" | Unassigned actions = nothing happens |
| "Recurring meetings are fine" | Each recurrence needs a reason; cancel if you're filling time |

## Red Flags
- No clear outcome statement
- Status updates that could have been a doc
- Same people talking; others silent for 30+ min
- Decisions made without the actual decider in the room
- No notes distributed after

## Verification
- [ ] Purpose + decider + pre-read verified before sending invite
- [ ] Agenda has outcome line and time-boxes
- [ ] Every action item has owner + deadline
- [ ] Notes sent within 24 hours
- [ ] Optional attendees actually made optional
